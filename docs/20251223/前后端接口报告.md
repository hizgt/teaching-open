# 前后端接口报告

## 项目概述

Teaching Open 教学管理系统接口文档，基于 GoFrame V2 + Vue3 技术栈。

**项目状态**: 后端开发已完成，前端开发准备就绪
**API版本**: v1.0
**生成时间**: 2025-12-23

## 接口规范说明

### 后端接口规范 (GoFrame V2)
- **基础路径**: `/api/v1`
- **响应格式**: 统一使用 `ghttp.Response` 包装
- **认证方式**: JWT Token (Authorization: Bearer {token})
- **分页格式**: `page` (页码), `size` (每页条数)

### 前端调用规范 (Vue3 + Axios)
- **HTTP库**: axios
- **基础配置**: baseURL = `/api/v1`
- **请求拦截器**: 自动添加 JWT Token
- **响应拦截器**: 统一错误处理和登录过期跳转

## 前端调用方式说明

### 1. 基础请求配置

```javascript
// src/utils/request.js
import axios from 'axios'

const request = axios.create({
  baseURL: '/api/v1',
  timeout: 10000,
  headers: {
    'Content-Type': 'application/json'
  }
})

// 请求拦截器 - 添加token
request.interceptors.request.use(config => {
  const token = localStorage.getItem('token')
  if (token) {
    config.headers.Authorization = `Bearer ${token}`
  }
  return config
})

// 响应拦截器 - 处理错误
request.interceptors.response.use(
  response => response.data,
  error => {
    if (error.response?.status === 401) {
      // 跳转登录页
      router.push('/login')
    }
    return Promise.reject(error)
  }
)

export default request
```

### 2. API调用示例

```javascript
// src/api/user.js
import request from '@/utils/request'

// 获取用户列表
export function getUserList(params) {
  return request({
    url: '/sys/user/list',
    method: 'get',
    params
  })
}

// 添加用户
export function addUser(data) {
  return request({
    url: '/sys/user/add',
    method: 'post',
    data
  })
}

// 更新用户
export function updateUser(data) {
  return request({
    url: '/sys/user/edit',
    method: 'put',
    data
  })
}

// 删除用户
export function deleteUser(id) {
  return request({
    url: '/sys/user/delete',
    method: 'delete',
    params: { id }
  })
}
```

### 3. 文件上传

```javascript
// 文件上传
export function uploadFile(file) {
  const formData = new FormData()
  formData.append('file', file)

  return request({
    url: '/sys/file/upload',
    method: 'post',
    data: formData,
    headers: {
      'Content-Type': 'multipart/form-data'
    }
  })
}
```

## 响应格式规范

### 成功响应

```json
{
  "code": 200,
  "message": "操作成功",
  "result": {
    // 数据内容
  },
  "success": true,
  "timestamp": 1640995200000
}
```

### 分页响应

```json
{
  "code": 200,
  "result": {
    "records": [],     // 数据列表
    "total": 100,      // 总条数
    "pageNo": 1,       // 当前页
    "pageSize": 10     // 每页条数
  },
  "success": true
}
```

### 错误响应

```json
{
  "code": 500,
  "message": "系统错误",
  "result": null,
  "success": false,
  "timestamp": 1640995200000
}
```

## 注意事项

1. **认证要求**: 除登录接口外，所有接口都需要JWT Token
2. **权限控制**: 基于RBAC模型，不同角色访问不同接口
3. **数据权限**: 部门数据隔离，教师只能看到本部门数据
4. **分页参数**: 统一使用 `page` (页码，从1开始) 和 `size` (每页条数)
5. **文件上传**: 支持单文件和批量上传，最大50MB
6. **缓存策略**: 字典数据使用Redis缓存，需要及时刷新
