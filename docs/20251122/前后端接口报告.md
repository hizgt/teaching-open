## 📊 统一响应格式

所有接口统一返回以下格式：

```json
{
  "code": 0,           // 错误码，0表示成功
  "message": "string", // 消息
  "result": {},        // 数据
  "success": true      // 是否成功
}
```

### 错误码说明

| 错误码 | 说明 |
|--------|------|
| 0 | 成功 |
| 1000 | 操作失败 |
| 1001 | 系统内部错误 |
| 1002 | 参数错误 |
| 2001 | 未授权 |
| 2002 | Token已过期 |
| 2003 | Token无效 |
| 2004 | 权限不足 |
| 2005 | 登录失败 |
| 2006 | 用户不存在 |
| 2007 | 密码错误 |
| 3001 | 记录不存在 |
| 3002 | 记录已存在 |

## 🔧 开发规范

### 请求头

所有需要认证的接口必须携带 Token：

```
X-Token: Bearer {token}
```

或

```
Authorization: Bearer {token}
```

### 分页参数

```
page: int        // 页码，从1开始
pageSize: int    // 每页数量，默认10，最大100
```

### 排序参数

```
sortField: string  // 排序字段
sortOrder: string  // 排序方向：asc/desc
```

### 时间格式

统一使用 RFC3339 格式：`2006-01-02T15:04:05Z07:00`

或简化格式：`2006-01-02 15:04:05`
