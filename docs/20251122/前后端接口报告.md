## 🔐 认证授权模块

### 用户登录

**接口**: `POST /sys/login`  
**状态**: ✅ 已完成  
**权限**: 公开  
**描述**: 用户登录接口,使用MD5+salt密码验证,返回JWT token

#### 请求参数
```json
{
  "username": "string",   // 用户名（必填）
  "password": "string"    // 密码（必填）
}
```

#### 响应示例
```json
{
  "code": 0,
  "message": "操作成功",
  "result": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "userInfo": {
      "id": "1",
      "username": "admin",
      "realname": "管理员",
      "avatar": "https://...",
      "status": 1,
      "school": "示例学校",
      "phone": "13800138000",
      "email": "admin@example.com",
      "sex": 1
    }
  },
  "success": true
}
```

#### 实现说明
- 密码验证: MD5(password + salt)
- Token有效期: 2小时
- 自动过滤已删除用户(DelFlag=1)
- 自动检查用户状态(Status: 1-正常, 2-冻结)

## 👤 用户管理模块

### 用户列表

**接口**: `GET /sys/user/list`  
**状态**: ✅ 已完成  
**权限**: 需要认证  
**描述**: 分页查询用户列表,支持模糊搜索和状态筛选

#### 请求参数
```
page: int        // 页码，默认1
pageSize: int    // 每页数量，默认10，最大100
username: string // 用户名（模糊查询）
realname: string // 真实姓名（模糊查询）
status: int      // 状态：1-正常，2-冻结
```

#### 响应示例
```json
{
  "code": 0,
  "message": "操作成功",
  "result": {
    "list": [
      {
        "id": "1",
        "username": "admin",
        "realname": "管理员",
        "phone": "13800138000",
        "email": "admin@example.com",
        "status": 1,
        "school": "示例学校",
        "sex": 1,
        "avatar": "https://..."
      }
    ],
    "total": 100,
    "page": 1,
    "pageSize": 10
  },
  "success": true
}
```

#### 实现说明
- 自动过滤已删除用户(DelFlag=0)
- 按创建时间倒序排列
- 不返回password和salt字段

### 添加用户

**接口**: `POST /sys/user`  
**状态**: ✅ 已完成  
**权限**: 需要认证

#### 请求参数
```json
{
  "username": "string",    // 用户名（必填，4-20字符）
  "realname": "string",    // 真实姓名（必填）
  "password": "string",    // 密码（必填，6-20字符）
  "phone": "string",       // 手机号（可选，需符合格式）
  "email": "string",       // 邮箱（可选，需符合格式）
  "school": "string",      // 学校
  "sex": 0                 // 性别：0-未知，1-男，2-女
}
```

#### 响应示例
```json
{
  "code": 0,
  "message": "操作成功",
  "result": {
    "id": "abc123",
    "username": "newuser",
    "realname": "新用户",
    ...
  },
  "success": true
}
```

#### 实现说明
- 自动生成UUID作为用户ID
- 自动生成8位随机salt
- 使用MD5(password + salt)加密密码
- 自动检查username/phone/email唯一性
- 默认状态: Status=1(正常), DelFlag=0(未删除)

### 编辑用户

**状态**: ✅ 已完成

#### 请求参数

```json
  "school": "string",      // 学校
  "sex": 0,                // 性别
  "status": 1              // 状态：1-正常，2-冻结
}
```

#### 响应示例
```json

  "success": true      // 是否成功
```

### 错误码说明

| 错误码 | 说明 |
|--------|------|
| 0 | 成功 |
| 1000 | 操作失败 |
| 1001 | 系统内部错误 |
| 1002 | 参数错误 |
| 2001 | 未授权 |
| 2002 | Token已过期 |
| 2003 | Token无效 |
| 2004 | 权限不足 |
| 2005 | 登录失败 |
| 2006 | 用户不存在 |
| 2007 | 密码错误 |
| 3001 | 记录不存在 |
| 3002 | 记录已存在 |

## 🔧 开发规范

### 请求头

所有需要认证的接口必须携带 Token：

```
X-Token: Bearer {token}
```

或

```
Authorization: Bearer {token}
```

### 分页参数

```
page: int        // 页码，从1开始
pageSize: int    // 每页数量，默认10，最大100
```

### 排序参数

```
sortField: string  // 排序字段
sortOrder: string  // 排序方向：asc/desc
```

### 时间格式

统一使用 RFC3339 格式：`2006-01-02T15:04:05Z07:00`

或简化格式：`2006-01-02 15:04:05`
