
**完成标准**: 能够执行 `gf` 命令

#### 任务2: 检查和完善项目结构
**当前状态**:
- ✅ 已有 `api-go/` 目录
- ✅ 已创建基本的目录结构
- ⏳ 需要完善配置文件

**需要创建的文件**:
1. `manifest/config/config.yaml` - 主配置文件
2. `hack/config.yaml` - DAO生成配置
3. `utility/response/response.go` - 统一响应格式
4. `utility/jwt/jwt.go` - JWT工具
5. `api/middleware/auth.go` - 认证中间件
6. `api/middleware/cors.go` - 跨域中间件

#### 任务3: 配置数据库连接
**配置内容**:
```yaml
database:
  default:
    link: "mysql:root:password@tcp(127.0.0.1:3306)/teachingopen"
    debug: true
    
redis:
  default:
    address: "127.0.0.1:6379"
    db: 0
```

#### 任务4: 生成 DAO 层代码
**步骤**:
1. 创建 `hack/config.yaml` 配置数据库信息
2. 执行 `gf gen dao` 生成代码
3. 验证生成的文件在 `internal/dao/`, `internal/model/do/`, `internal/model/entity/`

**预期生成表**:
- 系统管理: sys_user, sys_role, sys_permission, sys_depart, sys_dict 等
- 教学管理: teaching_course, teaching_work, teaching_student 等

#### 任务5: 实现统一响应格式
**文件**: `utility/response/response.go`

**功能**:
- `Success(ctx, data)` - 成功响应
- `Error(ctx, message)` - 错误响应
- `Json(ctx, code, message, data)` - 自定义响应
- `Page(ctx, data, total, pageNo, pageSize)` - 分页响应

#### 任务6: 实现 JWT 工具
**文件**: `utility/jwt/jwt.go`

**功能**:
- `GenerateToken(userId, username)` - 生成Token
- `ParseToken(tokenString)` - 解析Token
- `RefreshToken(tokenString)` - 刷新Token

#### 任务7: 实现基础中间件
**文件**:
- `api/middleware/cors.go` - CORS跨域处理
- `api/middleware/logger.go` - 请求日志记录
- `api/middleware/auth.go` - JWT认证验证

#### 任务8: 创建项目入口
**文件**: `main.go` 和 `internal/cmd/cmd.go`

**功能**:
- HTTP服务器启动
- 路由注册

- 配置加载

## 四、开发规范

### 4.1 代码规范

1. **命名规范**
   - 文件名: snake_case (如: sys_user.go)
   - 结构体: PascalCase (如: SysUser)
   - 方法名: PascalCase (如: GetUserList)
   - 变量名: camelCase (如: userName)

2. **注释规范**
   - 每个导出函数必须有注释
   - 注释格式: `// FunctionName 函数说明`
   - 复杂逻辑必须添加注释

3. **错误处理**
   - 使用 `gerror.New()` 创建错误
   - 错误信息清晰明确
   - 统一返回响应格式

### 4.2 Git 提交规范

**Commit 格式**:
```
<type>(<scope>): <subject>

<body>

<footer>
```

**Type 类型**:
- `feat`: 新功能
- `fix`: Bug修复
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建/工具变动

**示例**:
```bash
git commit -m "feat(auth): 实现JWT认证中间件"
git commit -m "fix(user): 修复用户列表分页bug"
git commit -m "docs(readme): 更新项目README"
```

### 4.3 测试规范

1. **单元测试**
   - 测试文件命名: `xxx_test.go`
   - 测试函数命名: `TestXxx`
   - 使用 `testing` 包

2. **接口测试**
   - 使用 Postman 或 Apifox
   - 保存测试集合
   - 提供测试数据
