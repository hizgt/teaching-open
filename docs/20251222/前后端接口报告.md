# 前后端接口报告

## 项目概述

本报告基于Jeecg Boot项目的前后端接口分析，整理了完整的接口列表，按模块分类展示。接口路径基于Spring Boot控制器注解定义，前端调用使用axios库，baseURL为`/api`。

## 接口规范说明

### 后端接口规范
- **基础路径**: `/api` (前端axios配置)
- **控制器路径**: 使用`@RequestMapping`类级注解定义
- **方法路径**: 使用`@RequestMapping`方法级注解定义
- **HTTP方法**: GET、POST、PUT、DELETE等
- **响应格式**: 统一使用`Result<T>`包装类

### 前端调用规范
- **HTTP库**: axios
- **基础配置**: baseURL = `/api`
- **请求拦截器**: 自动添加X-Access-Token
- **响应拦截器**: 统一错误处理

## 前端调用方式说明

### 1. 基础请求方法
前端使用统一的API调用方法：

```javascript
// GET请求
export function getAction(url, parameter) {
  return axios({
    url: url,
    method: 'get',
    params: parameter
  })
}

// POST请求
export function postAction(url, parameter) {
  return axios({
    url: url,
    method: 'post',
    data: parameter
  })
}

// PUT请求
export function putAction(url, parameter) {
  return axios({
    url: url,
    method: 'put',
    data: parameter
  })
}

// DELETE请求
export function deleteAction(url, parameter) {
  return axios({
    url: url,
    method: 'delete',
    params: parameter
  })
}
```

### 2. 文件上传
对于文件上传接口，使用FormData：

```javascript
export function uploadAction(url, parameter) {
  return axios({
    url: url,
    method: 'post',
    data: parameter,
    headers: {
      'Content-Type': 'multipart/form-data',
    },
  })
}
```

### 3. 响应格式
所有接口返回统一的Result格式：

```javascript
{
  success: true/false,  // 是否成功
  message: "操作成功",   // 提示信息
  result: {},           // 返回数据
  code: 200             // 状态码
}
```

## 注意事项

1. **权限控制**: 大部分接口需要相应的权限，权限通过Shiro注解控制
2. **数据权限**: 部分接口使用@PermissionData注解进行数据权限过滤
3. **缓存管理**: 字典等数据使用Redis缓存，修改时需要刷新缓存
4. **文件处理**: 文件上传下载有专门的文件服务处理
5. **分页查询**: 列表查询统一使用MyBatis-Plus分页
6. **导入导出**: Excel导入导出使用Jeecg框架的Poi组件
