## 🔐 认证授权模块

### 用户登录

**接口**: `POST /api/v1/sys/login`  
**状态**: ✅ 已完成  
**权限**: 公开  
**描述**: 用户登录接口，使用MD5+salt密码验证，返回JWT token

#### 请求参数
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名 |
| password | string | 是 | 密码 |

#### 请求示例
```json
{
  "username": "admin",
  "password": "123456"
}
```

#### 响应示例
```json
{
  "code": 0,
  "message": "操作成功",
  "result": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "userInfo": {
      "id": "1",
      "username": "admin",
      "realname": "管理员",
      "avatar": "https://...",
      "status": 1,
      "school": "示例学校",
      "phone": "13800138000",
      "email": "admin@example.com",
      "sex": 1
    }
  },
  "success": true
}
```

#### 实现说明
- 密码验证: MD5(password + salt)
- Token有效期: 2小时
- 自动过滤已删除用户(DelFlag=1)
- 自动检查用户状态(Status: 1-正常, 2-冻结)

### 添加用户

**接口**: `POST /api/v1/sys/user`  
**状态**: ✅ 已完成  
**权限**: 需要认证

#### 请求参数
| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| username | string | 是 | 用户名，4-20字符 |
| realname | string | 是 | 真实姓名 |
| password | string | 是 | 密码，6-20字符 |
| phone | string | 否 | 手机号，需符合格式 |
| email | string | 否 | 邮箱，需符合格式 |
| school | string | 否 | 学校 |
| sex | int | 否 | 性别：0-未知，1-男，2-女 |

#### 请求示例
```json
{
  "username": "newuser",
  "realname": "新用户",
  "password": "123456",
  "phone": "13800138001",
  "email": "newuser@example.com",
  "school": "示例学校",
  "sex": 1
}
```

#### 响应示例
```json
{
  "code": 0,
  "message": "操作成功",
  "result": {
    "id": "abc123",
    "username": "newuser",
    "realname": "新用户"
  },
  "success": true
}
```

#### 实现说明
- 自动生成UUID作为用户ID
- 自动生成8位随机salt
- 使用MD5(password + salt)加密密码
- 自动检查username/phone/email唯一性
- 默认状态: Status=1(正常), DelFlag=0(未删除)

## 📊 统一响应格式

所有接口统一返回以下格式：

```json
{
  "code": 0,           // 错误码，0表示成功
  "message": "string", // 消息
  "result": {},        // 数据
  "success": true      // 是否成功
}
```

### 分页响应格式

```json
{
  "code": 0,
  "message": "操作成功",
  "result": {
    "list": [],        // 数据列表
    "total": 100,      // 总数
    "page": 1,         // 当前页
    "pageSize": 10     // 每页条数
  },
  "success": true
}
```

### 错误码说明

| 错误码 | 说明 |
|--------|------|
| 0 | 成功 |
| 1000 | 操作失败 |
| 1001 | 系统内部错误 |
| 1002 | 数据库错误 |
| 1003 | 无效参数 |
| 2001 | 未授权 |
| 2002 | Token已过期 |
| 2003 | Token无效 |
| 2004 | 权限不足 |
| 2005 | 密码错误 |
| 2006 | 用户不存在 |
| 2007 | 用户已冻结 |
| 3001 | 记录不存在 |
| 3002 | 记录已存在 |
| 3003 | 用户名已存在 |
| 3004 | 手机号已存在 |
| 3005 | 邮箱已存在 |

## 🔧 开发规范

### 请求头

所有需要认证的接口必须携带 Token：

```
X-Token: Bearer {token}
```

或

```
Authorization: Bearer {token}
```

### 分页参数

```
page: int        // 页码，从1开始
pageSize: int    // 每页数量，默认10，最大100
```

### 排序参数

```
sortField: string  // 排序字段
sortOrder: string  // 排序方向：asc/desc
```

### 时间格式

统一使用 RFC3339 格式：`2006-01-02T15:04:05Z07:00`

或简化格式：`2006-01-02 15:04:05`
